# 面向路侧部署的单目3D交通感知系统 (板载部署版)

本项目是针对智能交通/车路协同中路侧相机部署的单目3D目标检测系统，为瑞芯微 **ELF-RV1126 开发板**（1.8 TOPS NPU, ARM Cortex-A7）定制与部署。本仓库仅包含板端部署运行所需的**最小可运行代码与核心模型**。

---

## 📂 文件清单与说明

```text
面向路侧部署的单目3D交通感知系统/
├── README.md                   # 本说明文件
├── main_pipeline.py            # 主入口：端侧 AI 视觉应用端到端完整流水线
├── rknn_infer.py               # NPU 推理实现模块（使用 RKNNLite 运行环境）
├── cpu_postprocess.py          # 后处理模块（包含向量化批量 SVD 几何解算、NMS）
├── image_enhance.py            # 预处理模块（CLAHE 增强与图像质量过滤，集成 NEON 加速）
├── traffic_analyzer.py         # 跟踪与分析模块（车流计数、车速估计、虚拟线圈）
├── visualization.py            # 可视化模块（多面板态势仪表盘渲染绘制）
├── model_N-RTM3D.py              # 模型定义（NPU 硬件友好型模型结构定义，用于 PC 仿真）
├── N-RTM3D-int8.rknn            # 预编译的量化 NPU 离线模型文件
├── run.sh                      # 一键运行测试脚本
├── video.mp4                   # 演示视频：系统在板卡端运行与可视化的完整效果演示录像
├── requirements.txt            # 板卡端依赖库清单
└── demo_100/                   # 100 帧连续视频实车测试集（含图像、标注与标定参数）
```

## 📺 演示视频

本仓库已附带实际板端部署运行效果的演示视频文件：[video.mp4](video.mp4)。

该视频录制于 ELF-RV1126 开发板连接 7 寸 LCD 屏的真实运行场景，直观展现了以下系统功能：
* **路侧三维框感知**：对密集车流的 3D 边界框（底/顶面投影与偏航角）的实时推理。
* **BEV 鸟瞰图转换**：实时估算并绘制车辆在三维道路地面上的相对平面位置。

* **智能交通统计**：包括过线车流计数、单车实时车速解算以及排队拥堵密度分析。

## 🚀 运行指南

### 1. 边缘板卡端运行 (NPU 加速)
确保板卡端已配置好显示屏幕，并连接至板卡。
```bash
# 激活开发板自带的elf-env环境即可
source /elf-env/bin/activate
# 若无此环境，则安装依赖库
pip install -r requirements.txt

#  一键运行 100 帧本地仿真测试（自动调用 NPU 推理并渲染仪表盘）
bash run.sh
```

### 2. 服务器/PC 端运行 (仿真模式)
在未搭载 NPU 芯片的服务器或 PC 端，可以运行 CPU/GPU 仿真测试：
```bash
# 启动服务器仿真模式
bash run.sh --server
```
